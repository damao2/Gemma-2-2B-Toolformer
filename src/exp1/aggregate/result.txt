从这张表里，大致可以看出几条比较清晰的规律：

1. d_new 越大，效果越好（但收益递减）
看 mean_dce_full（越小越好）和 mean_cos_full（越大越好）：

对于任意一个 percentile，从 32 → 64 → 128：
mean_dce_full 单调下降
p=0.25: 0.1334 → 0.1230 → 0.1125
p=0.50: 0.1911 → 0.1573 → 0.1349
p=0.75: 0.3594 → 0.2911 → 0.2456
mean_cos_full 单调上升
p=0.25: 0.6527 → 0.6822 → 0.7019
p=0.50: 0.6511 → 0.6814 → 0.7065
p=0.75: 0.5985 → 0.6406 → 0.6748
同时，mean_l2_full 也随 d_new 变大而减小。

结论：

增加 delta 特征数量确实持续降低重建误差、提升与微调模型的一致性。
但 64 → 128 的收益已经明显小于 32 → 64，有“收益递减”的趋势。
2. percentile 越大，效果越差（p=0.75 明显不如 0.25/0.5）
在固定 d_new 下比较三行：

mean_dce_full：p=0.75 总是最大（CE 损失最大），p=0.25 最小或接近最小。
mean_cos_full：p=0.75 总是最小，p=0.25 / 0.5 较高。
mean_l2_full：p=0.75 也始终明显更大。
说明：

选择更高 percentile（0.75） 的 base SAE 版本，对微调模型来说反而更难适配；
0.25 或 0.5 的 base 作为“起点”更适合作为你做 delta 微调的基底。
3. base-only vs base+delta 的对比非常极端
mean_dce_base 在 12–14 量级，而 mean_dce_full 只有 0.11–0.36 左右：
说明原始 Gemma-Scope SAE 单独用在你的微调模型上非常不适配；
加上 delta 后，几乎把这十几点 ΔCE 全部修复掉了。
mean_cos_base 只有 ~0.44–0.47，而 mean_cos_full 都在 ~0.60–0.71：
base-only 对微调模型 MLP 的重建方向差很多；
base+delta 显著拉近（但还没到 0.9+，说明还有空间）。
这很好地支持了你的设想：

“通过在 SAE 上加少量 delta features，可以把一个适配原始模型的 SAE 迁移到微调模型上，修复绝大部分失配。”

4. 哪个组合“最好”？
如果综合看 mean_dce_full 和 mean_cos_full：

最佳区域在：d_new=128 且 percentile=0.25 或 0.50：
p=0.25：mean_dce_full=0.1125, mean_cos_full=0.7019
p=0.50：mean_dce_full=0.1349, mean_cos_full=0.7065
p=0.75 系列整体偏差明显更大，可以认为是不太推荐使用的组合。
更细一点的结论可以写成：

如果更在意CE 误差最小：d_new=128, p=0.25 略占优。
如果更在意平均 cos 稍高一点：d_new=128, p=0.50 略占优。
在参数预算有限的情况下，d_new=64, p=0.25/0.50 是一个性价比较高的折中点（误差已经很低，cos 较高，delta 维度只有原来的一半）。
如果你接下来要写总结，可以把这一页说成：

baseline SAE 在微调模型上严重失配；
加入少量 delta 特征后，几乎完全修复；
更大的 d_new 单调改善，但收益递减；
选择较低的 percentile（0.25/0.5）作为 base 起点更有利于迁移，0.75 明显较差。
